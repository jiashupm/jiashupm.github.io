为什么要做用户画像？（目的）

  • 明确知道用户的真实诉求
  • 为功能设计提供核心依据
  • 数据挖掘与用户推荐的底层支持

    怎么做用户画像？

  • 流程：信息（处理、抽离、分析） –> 标签 –> 方案（优化产品、运营方案……）
  • 用户画像需要做统计的几个维度
    • 基础属性（人口学统计）：性别、年龄、教育、职业……
    • 社会关系：父母、子女、兄弟姐妹、同事……
    • 消费能力：收入、消费、信用卡、贷款……
    • 行为特征：经常团购、经常加班、经常熬夜、喜欢深夜听歌……
    • （心理特征：贪小便宜、好攀比、追求小众……

注意：用户的行为特征是用户特征最直接的反应。

  • 具体分析操作流程
    1. 个体分析，从以上5个维度分析用户特性
    2. 分析总结关键行为特征，明确特征的计算方法，即定义符合哪些行为特征的用户为目标用户
    3. 召回验证，使用上一步骤总结出的关键性行为特征导出数据，进行验证；查看召回率、召回准确率，步骤b得出的算法是否准确。并持续优化，调整关键行为粒度。
    4. 寻找解决方案

      小结

      用户画像其实不止一整个app的大用户画像，更可以将场景细化。比如深夜浏览的用户的用户画像、浏览某种内容的用户 用户画像等等，以便于产品/运营侧更准确的提出相应策略，进行产品优化。
      同时在运营侧，应用用户画像做用户分层，从而做到精细化、分层化运营。
